# Схема данных

## PostgreSQL (`deploy/init/postgres/001_schema.sql`)

```
users      (id, login UNIQUE, password_hash, created_at)
accounts   (id, user_id FK, currency, cash_balance CHECK >= 0, created_at)
instruments(id, symbol UNIQUE, name, dev_idx)
orders     (id, user_id, symbol, side, type, qty, price, filled_qty,
            status, client_order_id UNIQUE, created_at, updated_at)
trades     (id, order_id, user_id, symbol, side, qty, price, fee, executed_at)
positions  (user_id, symbol, qty CHECK >= 0, avg_price, PK(user_id, symbol))
```

Индексы: `orders(status, symbol)` — выборка активных лимиток при матчинге;
`orders(user_id, created_at DESC)`, `trades(user_id, executed_at DESC)` —
списки в приложении.

Инварианты обеспечивает БД + транзакции:
- баланс и количество в позиции не уходят в минус (CHECK);
- сделка/позиция/баланс меняются атомарно;
- повторный ордер с тем же `client_order_id` не создаётся (UNIQUE).

## ClickHouse (`deploy/init/clickhouse/001_ticks.sql`)

```sql
trading.ticks (symbol LowCardinality(String), ts DateTime64(3),
               price Decimal(18,6), volume UInt32, seq UInt64)
ENGINE = MergeTree ORDER BY (symbol, ts);

trading.candles_1m  -- AggregatingMergeTree, заполняется MV из ticks
(symbol, t, open argMin, high max, low min, close argMax, volume sum)
```
