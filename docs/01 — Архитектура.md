# Архитектура

```
Android (#1)        Load Simulator (#5)
      └──── REST + WebSocket ────┘
                  │
          Gateway #3 (Ktor :8080)
          JWT · rate limit · WS-стрим
            │                │
   Data-сервис #4         Redis ◄── Quotes #6 (Go :8081) ◄── драйвер C #7
   (Ktor :8082)          Pub/Sub        │                    /dev/stock0..3
    │        │             кэш цен      ▼
PostgreSQL  Redis                   ClickHouse (тики, свечи)
```

## Поток котировок (read path)

1. Драйвер `stock_gen.c` генерирует тики (GBM) в `/dev/stock0..3`
   (без ядра — встроенный симулятор в quotes-go, `QUOTES_SIMULATE=1`).
2. `quotes-go` читает 32-байтовые записи, маппит `dev_idx → тикер` и:
   - публикует JSON в Redis-канал `quotes:ticks`,
   - обновляет ключ `quote:last:<symbol>` (кэш последней цены),
   - батчами пишет в ClickHouse `trading.ticks` (1 с / 5000 строк).
3. Gateway держит **одну** подписку на `quotes:ticks` и раздаёт тики
   WebSocket-клиентам через `SharedFlow` (fan-out в памяти).
4. Свечи `candles_1m` строятся материализованным представлением ClickHouse;
   история для графиков агрегируется из тиков на лету.

## Поток торговли (write path)

1. Клиент → `POST /orders` на Gateway (JWT обязателен, `userId` из токена).
2. Gateway проксирует в Data-сервис с пробросом `traceparent`.
3. Data-сервис: market-ордер исполняется сразу по цене из Redis-кэша;
   limit сохраняется со статусом `NEW`.
4. `MatchingWorker` слушает `quotes:ticks` и исполняет пересечённые лимитки
   (`FOR UPDATE SKIP LOCKED`) по цене тика.
5. Сделка + пересчёт позиции + изменение баланса — **одна транзакция**;
   баланс защищён `CHECK (cash_balance >= 0)`; идемпотентность по
   `clientOrderId`.

## Ключевые решения

- **Redis Pub/Sub как шина тиков** — развязывает quotes-go, Gateway и
  Data-сервис; каждый потребитель независим.
- **Голый SQL** (требование) — репозитории на `Connection`, компонуются
  в транзакции, только параметризованные запросы.
- **Money-поля строками в JSON** — NUMERIC(18,6) не влезает в double без
  потери точности.
- **ClickHouse для истории** — MergeTree по (symbol, ts), свечи через
  AggregatingMergeTree MV.
