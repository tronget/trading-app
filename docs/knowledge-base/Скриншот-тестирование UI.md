# Скриншот-тестирование UI

Скриншот-тесты нужны, чтобы проверять не только логику Android-приложения, но и
то, как реально выглядит экран. Обычный unit-тест не поймает съехавший отступ
или пропавшую кнопку.

## Как работает проверка

Тест открывает экран, рендерит его в картинку и сравнивает с эталоном. Если UI
изменился случайно, тест упадёт. Если я изменил экран специально, то обновляю
эталон.

Это удобно для проекта с Compose, потому что экраны можно собрать из состояния
и проверить без ручного кликанья по приложению.

## Почему без эмулятора

В этом проекте запускать UI-тесты на эмуляторе неудобно. Он медленный, а на
машине, где собирался проект, ещё и падал из-за проблемы QEMU.

Поэтому используется Robolectric: Android-код запускается прямо на JVM. А
Roborazzi уже делает снимок экрана и сравнивает его с сохранённой картинкой.

## Какой рабочий процесс

Если экран изменён осознанно:

```bash
./gradlew recordRoborazziDebug
```

Если нужно проверить, что ничего не разъехалось:

```bash
./gradlew verifyRoborazziDebug
```

Эталоны лежат в репозитории. Их можно использовать и как доказательство в
отчёте: видно, какие экраны реально были собраны.
