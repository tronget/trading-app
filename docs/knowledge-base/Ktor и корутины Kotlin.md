# Ktor и корутины Kotlin

Gateway и Data-сервис написаны на **Kotlin + Ktor**. Для обработки запросов и
фоновых задач используются корутины.

## Корутины: асинхронность, которая читается как обычный код

Сервер постоянно кого-то ждёт: ответа от БД, от Redis, от другого сервиса. Если
на каждый запрос вешать **поток ОС** и блокировать его на ожидании — потоки
кончатся уже на паре тысяч клиентов (каждый поток весит мегабайты).

Корутина — это лёгкая задача, которую можно приостановить на время ожидания,
освободив поток для других задач, и возобновить позже. Тысячи
корутин крутятся на десятке потоков.

Ключевое слово здесь — `suspend`:

```kotlin
suspend fun place(request: PlaceOrderRequest): OrderDto {
    val order = db.tx("place_order") { … }   // тут корутина «засыпает», ждя БД,
    return order                              // но поток не блокируется
}
```

Код выглядит последовательно, без цепочек callback-функций. В точке
`db.tx(...)` корутина не держит поток, пока БД обрабатывает запрос.

JDBC — блокирующий API. Поэтому работа с БД выполняется через
`withContext(Dispatchers.IO)`. Так JDBC не занимает потоки, которые обслуживают
сетевые запросы.

## Structured concurrency

Корутины запускаются в **scope** и образуют дерево: родитель ждёт детей, а
отмена/ошибка распространяется по дереву. Нельзя «забыть» корутину — она привязана
к области видимости. Так
`MatchingWorker` живёт фоновой корутиной, пока работает приложение:

```kotlin
fun start(scope: CoroutineScope): Job = scope.launch {
    prices.ticks.collect { tick -> matchSymbol(...) }   // бесконечно слушает поток
}
```

## Ktor: сервер из плагинов

Функции Ktor подключаются через плагины `install(...)`. Основная конфигурация
Gateway выглядит так:

```kotlin
install(ContentNegotiation) { json(...) }   // (де)сериализация JSON
install(WebSockets) { pingPeriod = 30.seconds }
install(RateLimit) { … }                     // см. [[Rate limiting]]
install(Authentication) { jwt("jwt") { … } }
install(StatusPages) { … }                   // ошибки → аккуратный JSON

routing {
    get("/health") { call.respond(...) }
    rateLimit(RateLimitName("api")) {
        authRoutes(...); quoteRoutes(...)
        authenticate("jwt") { tradingRoutes(...) }  // только с валидным токеном
    }
}
```

`StatusPages` превращает исключения в понятные HTTP-ответы. Стектрейсы при этом
не попадают клиенту.

Kotlin используется и на сервере, и в Android-приложении. Это упрощает работу
с моделями и корутинами в разных частях проекта.
