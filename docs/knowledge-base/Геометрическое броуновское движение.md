# Геометрическое броуновское движение

Геометрическое броуновское движение используется в генераторе котировок. Цена
должна двигаться не случайным «плюс-минус рубль», а ближе к рыночному
поведению: иногда расти, иногда падать, но не уходить в отрицательные значения.

## Идея модели

Геометрическое броуновское движение моделирует не абсолютное изменение цены, а
процентное. Это логичнее для акций: бумага за 10 и за 1000 рублей обычно
двигается не на одинаковое число рублей, а примерно на доли своей цены.

Ещё в модели есть две настройки:

- общее направление движения;
- сила случайных колебаний.

Если колебания большие, график получается нервным. Если маленькие — почти
ровным. В проекте это удобно: можно менять параметры и получать разное
поведение котировок.

## Почему цена не становится отрицательной

Обычное случайное прибавление может случайно загнать цену ниже нуля. Для акции
это выглядит странно. В геометрической модели изменение умножается на текущую
цену, поэтому значение остаётся положительным.

Для учебного проекта мне не нужна идеальная финансовая модель. Нужно, чтобы
котировки выглядели правдоподобно и постоянно менялись.

## Техническая деталь

Один шаг модели можно записать так:

```text
price(t+1) = price(t) × exp(drift·dt + volatility·√dt·Z)
```

В драйвере это считается внутри ядра Linux, а там нельзя использовать обычные
числа с плавающей точкой. Поэтому цена хранится в целых значениях, а сложные
части формулы заменены приближениями.

Случайное число `Z` получается через сумму нескольких случайных байт:

```c
for (i = 0; i < 12; i++) {
    get_random_bytes(&b, 1);
    sum += b;
}
return ((sum - 1530) * 1000) / 255;
```

Точность здесь не научная, но для генератора котировок её хватает.
