# Геометрическое броуновское движение

Драйвер генерирует цену по модели **геометрического броуновского движения**.
Она даёт случайные изменения, которые зависят от текущей цены.

## Как работает модель

«Броуновское движение» — это как пылинка, которую случайно толкают молекулы:
она дрожит туда-сюда. Цена акции ведёт себя похоже, но с двумя поправками:

1. Колеблется не сама цена, а её **процентное изменение**. Акции за $10 и за
   $1000 двигаются на проценты, а не на одинаковые центы. Отсюда «геометрическое».
2. Есть небольшой общий сдвиг вверх или вниз. В нашей конфигурации цена в
   среднем растёт.

Формула одного шага:

```
price(t+1) = price(t) × exp( drift·dt  +  volatility·√dt·Z )
```

- `dt` — кусочек времени (доля торгового года);
- `drift` — среднее направление изменения цены, у нас 5 % годовых;
- `volatility` — сила колебаний цены, у нас 20 % годовых;
- `Z` — случайное число из **нормального распределения** N(0,1), это и есть
  «толчок молекулы».

Ключевое свойство: цена под `exp(...)` **никогда не уходит в минус** (акция не
может стоить −$5), и шаги пропорциональны текущей цене. Именно это и нужно.

При положительном `drift` цена в среднем растёт. Параметр `volatility`
определяет силу колебаний. Оба значения можно менять через `ioctl`.

## Где взять случайность и почему всё в целых числах

Код считает GBM **внутри ядра Linux**, а
там **запрещено пользоваться числами с плавающей точкой** (FPU не сохраняется в
контексте ядра). А в формуле сплошные `exp`, `√`, проценты…

Решения два:

**1. Нормальное число `Z` — через центральную предельную теорему.** Честный
Box–Muller требует `log` и `sqrt` — нельзя. Поэтому код берёт 12 случайных
байт (каждый — равномерное число 0…255) и складывает. ЦПТ говорит: сумма
многих равномерных величин ≈ нормальное распределение. Сумму нормируют — и
получают приближённое `Z` (масштабированное на 1000, чтобы остаться в целых):

```c
for (i = 0; i < 12; i++) { get_random_bytes(&b, 1); sum += b; }
return ((sum - 1530) * 1000) / 255;   // ≈ Z·1000, N(0,1)
```

(1530 = среднее суммы 12 байт, 255 ≈ её стандартное отклонение.)

**2. Вся арифметика — фиксированная точка в `s64`.** Цена хранится в
**микродолларах** (целое число = доллары × 1 000 000), `exp(x)` для маленьких
`x` заменён линейным приближением `1 + x`, а `√dt` — табличкой готовых
значений для типичных интервалов. Точность «не для научной статьи, а чтобы цена
двигалась реалистично» — и этого достаточно.

Для учебного генератора такой точности достаточно. Цена остаётся положительной
и выглядит достаточно реалистично, а вычисления не используют FPU.
