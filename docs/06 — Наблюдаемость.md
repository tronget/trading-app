# Наблюдаемость

## Трейсы (OpenTelemetry → Jaeger)

Все сервисы включают экспорт, когда задан `OTEL_EXPORTER_OTLP_ENDPOINT`
(в docker-compose он указывает на otel-collector, тот пересылает в Jaeger).

Сквозной trace-id: Gateway создаёт серверный спан на каждый HTTP-запрос и
**инжектирует W3C `traceparent`** в вызовы Data-сервиса; Data-сервис
извлекает его и вешает дочерние спаны на каждую SQL-транзакцию
(`db.tx place_order`, `db.tx match_limit_orders`, …). Контекст в корутинах
переносится через `opentelemetry-extension-kotlin` (`asContextElement`).

quotes-go ставит спаны на публикацию в Redis (`redis.publish_tick`)
и батч-вставку в ClickHouse (`clickhouse.flush_ticks`).

Jaeger UI: http://localhost:16686 (сервисы `gateway`, `data-service`, `quotes`).

## Метрики

`quotes-go` отдаёт `/metrics` в формате Prometheus:

```
quotes_ticks_total            счётчик прочитанных тиков
quotes_ticks_per_second       гейдж (окно 1 с)
quotes_ticks_dropped_total    переполнение буферов синков
quotes_redis_published_total / quotes_redis_errors_total
quotes_clickhouse_rows_total / quotes_clickhouse_errors_total
```

Нагрузочные метрики (p50/p95/p99, RPS, ошибки) считает имитатор.

## Логи

Kotlin-сервисы пишут структурированный JSON через logback
(`{"ts":...,"level":...,"logger":...,"message":...}`), quotes-go — стандартный
лог с латентностью каждого HTTP-запроса. Ошибки Redis/ClickHouse в quotes-go
троттлятся (не чаще раза в 5 с), чтобы не заливать лог при недоступности.
